先把真实波动看清楚，再接「电话叫醒」。

这是一个静态 H5 演示站：页面会读取部署时写入的 quotes/last.json（由 GitHub Actions 从 Yahoo 拉取），并用你在本页设置的本地阈值即时判断是否越线；同时嵌入 TradingView 作为对照。本站不会、也无法发起真实电话——外呼必须跑在你自己的后端服务里并完成运营商侧配置。

  • 本地阈值越线提示（不通知、不拨号）

典型使用场景

不是让你一直盯图，而是在关键时刻把你叫回来。

上班通勤

你在路上、会议中、或不方便看盘时，先用快照与越线提示判断是否需要回到屏幕前；若已自建告警服务，再由服务端触发电话/推送。

夜盘跨时区

用北京时间理解自己的节奏，用纽约时间确认市场状态，避免时差和休市判断带来的误解。

风控兜底

当 SPY 进入大波动区间，系统自动补位，帮你把“看见波动”变成“及时行动”。

移动端体验

一切交互都围绕手机单屏完成。

单手操作路径（本页已实现）

首屏即可：刷新快照、调整本地阈值、对照 TradingView。按钮与留白按 H5 点击热区处理。

  • 首屏展示快照涨跌幅、最新价/昨收、距离阈值
  • 阈值滑块仅影响浏览器内的提示逻辑
  • 嵌入第三方行情组件用于肉眼核对
  • 深色背景降低夜间查看刺激感

工作流程

从「看得见波动」到「叫得醒人」，中间差一个后端。

01

静态页：快照 + 阈值提示

你现在看到的就是这一层：GitHub Actions 写入 JSON，浏览器读取并计算是否越线。

02

服务端：轮询、去重、风控

需要常驻进程处理行情源失败、阈值抖动、冷却窗口、以及外呼频控——这些都不适合放在纯静态托管里。

03

语音通道：Twilio 等 + 公网回调

真正拨号需要服务商账号、签名验证与可访问的回调 URL。把这部分放在服务器上，H5 只作为配置与状态面板。
